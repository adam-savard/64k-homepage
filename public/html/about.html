About the 28k Homepage Project


But... why?

People these days don't like to wait around for a webpage to load. If it takes 2-3 seconds, we already lose our patience.
If we take longer than 5 seconds, we're prone to click away. The 28k-homepage project aims to correct that for older machines.

Using a 9600 baud modem, 3 seconds can transfer a maximum of 28800 bytes, given ideal conditions. So, with that in mind,
I set out to limit the web to something that dial-up users would be proud of.
So you want to make yourself a homepage? Neat. Let's talk about some of the limitations:

  • Webpage must be 28.8k or less
  • No JavaScript (script tags are parsed out)
  • Images must be hosted off-site
  • A single html file can be submitted


And here are some guidelines (optional):
  • Retro-friendly means 640x480 or less
  • Images should not be massive and should be 256 colors or less
  • Limited CSS; remember that older machines can't handle HTML5


Doesn't seem too limiting, does it? This is all in an effort to present information as consicely and fast as possible. The goal for the whole page is to be retro-friendly, so that an 8088 could view the page.

As for content, you can put whatever you like, though the idea is to create homepages to share with others; think of it like a landing page that describes a topic, yourself, etc.
